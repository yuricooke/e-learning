O que é R?

R é pacote open-source de software para Análise de Dados e Estatística. Com uma comunidade bastante ativa, atinge a ordem de centenas de milhares de usuários, hoje.

Foi criado em 1993 por Ross Ihaka e Robert Gentleman, do Departamento de Estatística da Universidade de Auckland, na Nova Zelândia, como uma implementação distinta de um produto da AT&T, antiga Bell Laboratories, o "S".

Hoje é mantida sobretudo pela R Foundation for Statistical Computing, com sede em Viena. Em relação a outros produtos que processam datasets em RAM, a performance do R é comparável, ou superior.

FUNÇÕES NO R

No R, uma função é um bloco de instruções organizadas de maneira a realizar uma tarefa específica. O R tem um grande número de funções integradas, mas o usuário pode criar suas próprias funções.

Em R, uma função pode ou não precisar de argumentos de entrada, podendo também retornar, ou não, alguma saída.

Uma função R é criada utilizando-se o comando function. Já a instrução return() retorna um valor ou objeto, quando é o caso. A sintaxe básica para a definição de uma função em R é a seguinte:

    
1
2
3
4
5
nome_função <- função (arg_1, arg_2, ...) {

Corpo da função

return(valor_retornado) }

As diferentes partes de uma função são:

nome_funcao 

Este é o nome real da função. Ele é armazenado no ambiente R como um objeto com este nome.

arg_1, arg_2...

São os argumentos da função. Quando uma função é chamada, você normalmente passará valores de entrada como argumentos, respeitando a ordem em que foram estipulados quando da construção da função ou usando os nomes destes argumentos, explicitamente. Nem sempre uma função demandará argumentos. Além disso, os argumentos podem ter valores padrão.

corpo da função

Consistirá no conjunto de instruções, ou trecho de programa, ou subrotina, que define o que a função faz.

valor_retornado

O valor de retorno de uma função, quando existe, é retornado quando da execução do comando return(), o que também encerra a execução da função.

Estruturas de Dados definem formas específicas de organizar e armazenar dados.

As Estruturas de Dados em R podem ser organizadas segundo a sua dimensionalidade ou a sua heterogeneidade.

Não existem estruturas 0-dimensionais (escalares) no R. Todas as estruturas de dados do R derivam das primitivas abaixo:

Homogêneas Heterogêneas
1 - Dimensional Vetores “atômicos” Listas
2 - Dimensional Matrizes (Arrays 2D) Dataframes
n - dimensional Arrays

VETORES ATÔMICOS

Vetores atômicos são estruturas de dados homogêneas e unidimensionais. Todos os elementos do vetor serão do mesmo tipo de dados.

Um vetor atômico pode ser atribuído a uma variável, ou pode constituir em elemento de uma outra estrutura de dados, como listas.

COERÇÃO

Por vezes, é necessário tratar um dado considerando-o de um tipo que não seria o originalmente atribuído a ele. E isso chamamos coerção de tipo.

No R, coerções de tipo podem ocorrer de forma implícita, operada pelo R de forma automática em função de algum tipo de operação em curso, ou explícita, quando usuário especifica quando deve ocorrer uma coerção de tipo, e qual será o tipo de dado "de chegada".

Exemplo:

    
1
2
3
4
booleanos <- c(TRUE, FALSE, FALSE)
num <- as.integer(booleanos)
num
[1] 1 0 0

Aqui a coerção foi explícita e induziu tomar o valor TRUE como 1.0 e FALSE como 0.0.

ARRAYS E MATRIZES

Clique nas caixas abaixo e saiba o significado.

Arrays são as estruturas de dados que implementam "hiperparalelepípedos", porque organizam-se em estruturas de N dimensões. Por exemplo: um array de dimensões (2, 3, 4) consistiria em 4 matrizes, cada uma com 2 linhas e 3 colunas. Os arrays são estruturas homogêneas: podem armazenar apenas um tipo de dado.

Um array é criado usando-se a função array().

Já Matrizes são objetos nos quais os elementos são organizados em um layout retangular bidimensional. As matrizes são homogêneas, contêm elementos de um mesmo tipo.

Uma matriz é criada usando a função matrix().

ESTRUTURAS HOMOGÊNEAS ARRAYS E MATRIZES

LISTAS

Listas são estruturas heterogêneas do R que podem conter elementos de diferentes tipos, como: números, strings, vetores e outras listas. Uma lista também pode conter matrizes ou funções como seus elementos. A lista é criada usando a função list() .

Criação de uma lista

A seguir, um exemplo para criar uma lista contendo strings, números, vetores e valores lógicos.

print(list_data)

ESTRUTURAS HETEROGÊNEAS | LISTAS

DATAFRAMES

Um Dataframe é uma estrutura heterogênea bidimensional, organiza-se em campos e registros, podendo cada campo conter um tipo de dado. Podem ser entendidos como uma implementação de "tabelas".

A seguir estão as características de um Dataframe:

Os nomes das colunas não devem estar vazios.

Os nomes das linhas devem ser exclusivos.

Os dados armazenados em um dataframe de dados podem ser numéricos, textuais ou booleanos.

Cada coluna deve conter o mesmo número de itens de dados.

CONSIDERAÇÕES FINAIS

O R é uma numérica livre, gratuita e sob uma comunidade madura, crescente e ativa, para Computação aplicada à Estatística. R é usado em Ciência de Dados nos domínios os mais diversos, por empresas, meios acadêmicos e entusiastas.

Não é uma “bala de prata”, mas é potente, versátil e portável a ponto de poder ser incorporado a outros sistemas, complementando suas funcionalidades, por exemplo.

As estruturas de dados são variadas (listas, matrizes, dataframes etc.), sendo o vetor, o “átomo”. Por ser “programável”, garante a reprodutibilidade de tarefas de processamento e análise.

No concernente à visualização de dados, dispõe de uma miríade de componentes variados, amplamente customizáveis.

No R, as estruturas de dados podem ser construídas a partir de outras.

Há estruturas primitivas, a saber: vetores “atômicos”, arrays, listas e dataframes.

Os tipos primitivos de dados, no R, são: inteiro, número de dupla precisão, lógico (“booleano”), string, raw e complexo. Os valores podem sofrer coerção de tipo, deliberadamente ou por conta da operação em curso.

Uma vez organizados em uma estrutura conveniente, os dados podem ser manipulados como se queira, visando um certo resultado.
